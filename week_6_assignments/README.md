React State Management, Context API & MERN Stack

This week focuses on advanced React concepts including state management, Context API, and building a full-stack MERN application.

`User-form-with-dynamic-table` is a React app (Vite + React Router) that implements a user registration form with dynamic table rendering — users can submit data through a form and see it displayed in a live-updating table, demonstrating controlled components and state-driven UI.

`context_counter` is a React application deployed on Vercel that implements a counter using React's Context API for global state management, allowing multiple components to share and update a common counter state without prop drilling.

`counter-state-with-sync-context-store` extends the counter concept by combining React state with Context API and a synchronized store pattern, demonstrating more advanced state management strategies for maintaining consistent state across the component tree.

`user_count_react` is a Vite-powered React project that tracks and displays user counts, practicing `useState`, `useEffect`, and component lifecycle management for real-time data updates.

`mern-mini-employee-app` is a full-stack MERN (MongoDB, Express, React, Node.js) application with separate `backend` and `frontend` directories. It implements an employee management system with CRUD operations, connecting a React frontend to an Express/MongoDB backend API.

This week marks the transition from frontend-only development to building complete full-stack web applications.
